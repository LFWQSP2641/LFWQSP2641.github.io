绕过校园网认证

本文仅供研究和测试，请勿将其用于任何收费或违法活动，后果自负。

原理

在未进行校园网认证的情况下，通常不会屏蔽 DNS 查询（53 端口）。因此，可通过在 53 端口运行 Hysteria 2 或基于 UDP 的 mKCP 来建立通信。

准备工作

需要一台远程服务器，不建议使用国外服务器。UDP 拥有明显的流量特征，mKCP 的加密方式也易被识别，可能导致服务器 IP 被封禁。

所需条件

  • 未认证时不封锁 53 端口的校园网
  • 一台远程服务器（Linux 或 Windows 均可，这里以 Linux 为例）
  • 支持 Hysteria 2 或 vmess 的客户端，比如：
    • Windows
      • v2rayN
      • nekoray
    • Android
      • v2rayNG
      • NekoBox
